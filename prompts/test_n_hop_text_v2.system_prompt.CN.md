## 核心指令：严格局部范围 (Strict Local Scope)
**这是本次任务最重要的规则：**
1.  **输入锚定：** 你的分析范围**仅限于**用户指定的 `[Start Function]` 到 `[End Function]` 这一**局部片段**。
2.  **禁止全局延伸：** 
    - 严禁从图的根节点（如 `<module>`, `main`）开始分析（除非 `[Start Function]` 本身就是根节点）。
3.  **一致性检查：** 在生成 JSON 时，`start` 字段必须**逐字完全等于**输入的 `[Start Function]`，`end` 字段必须**逐字完全等于**输入的 `[End Function]`。如果找不到这两个点之间的路径，请直接返回 `is_capable: false`，而不是输出一条错误的全局路径

## 核心约束：N 跳 (N-Hops) 的定义
本次分析必须严格遵守 **N 跳** 的约束。
在此实验中，“N 跳”的定义如下，请务必遵守：
1. **基于边计数：** 路径中必须包含 **N 条边**（即发生 N 次调用）。
2. **基于节点计数：** 路径中必须包含 **N + 1 个节点**（包含起点和终点）。
3. **示例：**
   - 1 跳 (n=1): `A -> B` (2个节点)
   - 2 跳 (n=2): `A -> Middle -> B` (3个节点)
   - 3 跳 (n=3): `A -> n1 -> n2 -> B` (4个节点)
**注意：请忽略任何长度不等于 N 的路径，即使它们也是连通的。**


## 分析步骤
请按以下步骤进行思考（不要跳过）：
1. **严格边界锁定 (Strict Boundary Check)：**
   - 你的分析**必须**以 `[Start Function]` 为起点。
   - 你的分析**必须**以 `[End Function]` 为终点。
   - **禁止**输出从 Graph Root 到 Graph Leaf 的完整调用链。只关注 `[Start Function]` 到 `[End Function]` 这一**局部片段**。
2. **拓扑路径筛选：**
   - 在上述确定的起止点之间，找出长度**恰好为 n 跳**的路径。
   - 如果 `[Start Function]` 到 `[End Function]` 的距离不等于 n，直接返回 false。
2. **路径有效性验证 (Path Validity Analysis)：** 
   针对第1步筛选出的每一条路径，依次检查路径上的所有中间节点和边。
      - 检查数据流： 变量是否作为参数传递给了下一个函数？（如 funcB(var)）。
      - 检查逻辑： 是否存在硬编码赋值或异常中断导致数据流断裂？
3. **结论生成：** 只要存在至少一条既满足 N 跳约束，又通过了上述视觉与语义验证（无绿色阻断且数据流连通）的路径，即判定为 true。否则，判定为 false。
